Edit Finding
Modify the finding details and save your changes.
  • Title: A concise, actionable description of the vulnerability.
  • Asset: The in-scope asset where this issue was found.
  • Severity: Rate from Critical to Info based on impact.
  • Status: Update the triage state — Open, Triage, Accepted, Fixed, or False Positive.
  • Assessment Area: High-level domain (e.g. Web Application, Network, Cloud).
  • OWASP Top 10: Map the finding to an OWASP 2021 category.
  • CWE: Search and select the relevant Common Weakness Enumeration entry.
  • Details: Rich markdown editor for the finding description. Type / for slash commands. Use /image to upload screenshots, or paste from clipboard.
  • Recommendation: Separate markdown editor for remediation guidance. Supports the same slash commands and image features.
  • Publish: If the finding is a draft, use the Publish button to finalize it. All required fields must be valid to publish.

New Finding
Create a security finding linked to an in-scope asset. The Statement of Work must be approved before findings can be created.
  • Title: A concise, actionable description of the vulnerability.
  • Asset: The in-scope asset where this issue was found. Only assets added to the SoW scope are available.
  • Severity: Rate from Critical to Info based on impact.
  • Status: Triage state — Open, Triage, Accepted, Fixed, or False Positive.
  • Assessment Area: The domain or area of the assessment (e.g. Web Application, Network, Cloud).
  • OWASP Top 10: The OWASP 2021 risk category this finding falls under.
  • CWE: Search and select the Common Weakness Enumeration identifier. Type to filter the list.
  • Details: Rich markdown editor for the finding description. Type / for slash commands. Use /image to upload screenshots, or paste from clipboard.
  • Recommendation: Separate markdown editor for remediation guidance. Supports the same slash commands and image features.
  • Save as Draft: Save an incomplete finding without requiring all fields. Publish it later from the edit screen.
